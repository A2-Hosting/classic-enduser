Hooks/define routes
===================

<div id="contentSub">

<span class="subpages">&lt; [Hooks](../Hooks.html "Hooks")</span>

</div>

<div id="primary">

This hook runs when Omeka's router is instantiated. It allows the plugin
writer to add routes to Omeka.

Routes can be used to create custom URLs that are different than the
default ones provided by Omeka for plugin pages.

Omeka uses the Zend Framework's routing functionality. For more
information on routes and Zend Framework, see Zend's
[documentation](http://framework.zend.com/manual/en/zend.controller.router.html).

<span id="Arguments" class="mw-headline"> Arguments </span>
-----------------------------------------------------------

 *[Zend\_Controller\_Router\_Rewrite](http://framework.zend.com/manual/en/zend.controller.router.html)* `$router` {{\#if
:   | (optional)}}
:   Router object.
:   {{\#if: | Default: `{{{default}}}`}}
